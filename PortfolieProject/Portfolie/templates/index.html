Online Chatroom with realtime message updating

This project was part of my third semesters mobile development exam where we had to make a program of our own choice. I made an online chatroom using Android Studio with a Firestore backend database.

The project estimation took me around 2 weeks and is only made using Java

How the app works is that the moment you launch it will ask for you name and log you in using that name with the help of the firestore. As you can see on the images there is currently only one chatroom that anybody can use, and write inside as they please, where the newest message is showed on the top.


When somebody adds a message it gets added to their own firestore storage collection, this helps keep track of who sent what, with one of the collections being the main chat called "messages". It has an extra variable that contains the time when every message was sent, so we can easily order the messages and have the newest one in the top.

Exam Project - Social Vibes messaging app

For my final exam of my Computer Science education, i got tasked with 2 other people to make a extension of the company "Social Vibes"s mobile app that could make events on a google api map and working messaging services between multiple users. The company had requirments in terms that the program needed to be worked on, such as flask python for the backend and Typescript for the frontend. Besides that we had full creativity control codewise and designwise.

This program was a 10 weeks project and was made using the requested tools, and for the management and storage of data given and recieved we have used Firebase's Firestore storage.

Since the program is an extension of our customers app we implemented a simple start loging to diverse the diffrent users and the chats/events they have created.

When the user is then logged in they get the option to create a new chat with the + in the corner or check on of their already existing chats. How that would work is just an automatic check depending how many people you would make a chatroom with that determines if its a "chat" or a "groupchat". Where the last thing is the community chat which was at this point made by the admins of the app [The Social Vibes team], and worked as an extension of a groupchat with more features inside it.

For the second main feature implemented which was the function to create custom events, where you had to be able to click on a google implemented map that would take the information and save it on the database in your name with all the information given.

Important thing to know is that majority of the frontend information to work, you need to run the backend code.
